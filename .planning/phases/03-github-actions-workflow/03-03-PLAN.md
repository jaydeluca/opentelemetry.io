From .github/workflows/collector-sync.yml:
```yaml
name: Sync Collector component docs

on:
  workflow_dispatch:
  schedule:
    - cron: '0 3 * * *'  # Daily at 03:00 UTC

permissions: read-all

jobs:
  sync-collector-docs:
    name: Sync Collector component documentation
    runs-on: ubuntu-latest
    permissions:
      contents: write
      pull-requests: write
      issues: write
    if: github.repository == 'open-telemetry/opentelemetry.io'
    steps:
      - uses: actions/checkout@v6
      - uses: actions/setup-python@a309ff8b426b58ec0e2a45f0f869d46889d02405 # v6.2.0
      - uses: astral-sh/setup-uv@5a095e7a2014a4212f075830d4f7277575a9d098 # v7.3.1
        with:
          enable-cache: true
      - uses: actions/setup-node@v6
        with:
          node-version-file: .nvmrc
          cache: 'npm'
          cache-dependency-path: package.json
      - name: Use CLA approved github bot
        run: |
          git config user.name otelbot
          git config user.email 197425009+otelbot@users.noreply.github.com
      - uses: actions/create-github-app-token@29824e69f54612133e76f7eaac726eef6c875baf # v2.2.1
        id: otelbot-token
        with:
          app-id: ${{ vars.OTELBOT_DOCS_APP_ID }}
          private-key: ${{ secrets.OTELBOT_DOCS_PRIVATE_KEY }}
      - run: bash .github/scripts/collector-sync.sh
        env:
          GH_TOKEN: ${{ steps.otelbot-token.outputs.token }}
          GITHUB_REPOSITORY: open-telemetry/opentelemetry.io
```

  <action>
    Create `.github/workflows/declarative-configuration-sync.yml` following collector-sync.yml pattern with two jobs:

    **Workflow metadata:**
    ```yaml
    name: Sync Declarative Configuration docs

    on:
      workflow_dispatch:
      schedule:
        # At 03:00 UTC, every day
        - cron: '0 3 * * *'

    # Declare default permissions as read only.
    permissions: read-all
    ```

    **Job 1: sync-lang-status**
    - Job name: "Sync language implementation status"
    - Runs on: ubuntu-latest
    - Permissions: contents: write, pull-requests: write (no issues: write needed)
    - Conditional: `if: github.repository == 'open-telemetry/opentelemetry.io'`

    Steps for sync-lang-status:
    1. Checkout code (actions/checkout@v6)
    2. Set up Python (actions/setup-python@a309ff8b426b58ec0e2a45f0f869d46889d02405 # v6.2.0)
    3. Install uv (astral-sh/setup-uv@5a095e7a2014a4212f075830d4f7277575a9d098 # v7.3.1 with enable-cache: true)
    4. Set up Node.js (actions/setup-node@v6 with node-version-file: .nvmrc, cache: npm, cache-dependency-path: package.json)
    5. Use CLA approved github bot (git config user.name otelbot, email 197425009+otelbot@users.noreply.github.com)
    6. Create otelbot GitHub App token (actions/create-github-app-token@29824e69f54612133e76f7eaac726eef6c875baf # v2.2.1, id: otelbot-token, vars.OTELBOT_DOCS_APP_ID, secrets.OTELBOT_DOCS_PRIVATE_KEY)
    7. Sync documentation (run: bash .github/scripts/declarative-configuration-sync.sh --mode=lang-status, env: GH_TOKEN from otelbot-token, GITHUB_REPOSITORY: open-telemetry/opentelemetry.io)

    **Job 2: sync-types**
    - Job name: "Sync types documentation"
    - Same structure as sync-lang-status
    - Final step runs: bash .github/scripts/declarative-configuration-sync.sh --mode=types
    - No job dependency (both jobs run independently)

    **Important details:**
    - Use exact SHA hashes for all actions (as shown in collector-sync.yml)
    - Include version comments after SHA (e.g., # v6.2.0)
    - Match indentation and formatting to collector-sync.yml
    - No concurrency settings (allow multiple runs to queue)
